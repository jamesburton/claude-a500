# Technical Spec: Motorola 68000 CPU Emulation

## Overview

The 68000 is the main processor. Emulation must be cycle-approximate for correct Amiga behavior (chipset timing depends on CPU bus contention).

## Architecture

### Register File

```csharp
// Core CPU state
uint[] D = new uint[8];     // D0–D7 data registers
uint[] A = new uint[8];     // A0–A7 address registers (A7 = active SP)
uint PC;                     // Program counter
ushort SR;                   // Status register
uint USP;                    // User stack pointer (saved when in supervisor mode)
uint SSP;                    // Supervisor stack pointer (saved when in user mode)
```

### Status Register Layout

```
Bit 15: T1 (Trace)
Bit 14: T0 (unused on 68000, always 0)
Bit 13: S  (Supervisor/User mode)
Bit 12-11: unused
Bit 10-8: I2-I0 (Interrupt mask, 0-7)
Bit 7-5: unused
Bit 4: X (Extend)
Bit 3: N (Negative)
Bit 2: Z (Zero)
Bit 1: V (Overflow)
Bit 0: C (Carry)
```

## Instruction Decoding

### Decode Strategy: Lookup Table

Use a 65536-entry dispatch table indexed by the first instruction word:

```csharp
delegate int OpcodeHandler(ushort opcode);
OpcodeHandler[] dispatchTable = new OpcodeHandler[65536];

// Build table at init time
void BuildDispatchTable()
{
    // For each possible 16-bit value, determine the instruction
    // and bind the appropriate handler with pre-decoded parameters
    for (int op = 0; op < 65536; op++)
    {
        dispatchTable[op] = DecodeOpcode((ushort)op);
    }
}
```

| Bits 15-12 | Group | Instructions |
|------------|-------|-------------|
| 0000 | Immediate/Bit | ORI, ANDI, SUBI, ADDI, EORI, CMPI, BTST, BCHG, BCLR, BSET, MOVEP |
| 0001 | MOVE.B | MOVE.B source,dest |
| 0010 | MOVE.L | MOVE.L source,dest / MOVEA.L |
| 0011 | MOVE.W | MOVE.W source,dest / MOVEA.W |
| 0100 | Miscellaneous | LEA, PEA, CLR, NEG, NOT, TST, SWAP, EXT, MOVEM, JSR, JMP, CHK, TRAP, LINK, UNLK, RTS, RTE, etc. |
| 0101 | ADDQ/SUBQ/Scc/DBcc | Quick add/sub, Set/Decrement-and-Branch on condition |
| 0110 | Bcc/BSR/BRA | Branch on condition, subroutine call, unconditional branch |
| 0111 | MOVEQ | Move quick (8-bit immediate to register) |
| 1000 | OR/DIV/SBCD | OR, DIVU, DIVS, SBCD |
| 1001 | SUB/SUBX | SUB, SUBA, SUBX |
| 1010 | (Line-A) | Unimplemented — trap |

## Addressing Modes

### Mode/Register Encoding (6 bits: mode[5:3], register[2:0])

| Mode | Reg | Syntax | Description |
|------|-----|--------|-------------|
| 000 | Dn | Dn | Data register direct |
| 001 | An | An | Address register direct |
| 010 | An | (An) | Address register indirect |
| 011 | An | (An)+ | Post-increment |
| 100 | An | -(An) | Pre-decrement |
| 101 | An | d16(An) | Displacement (16-bit signed) |
| 110 | An | d8(An,Xn) | Index (8-bit displacement + index register) |
| 111 | 000 | addr.W | Absolute short (sign-extended) |
| 111 | 001 | addr.L | Absolute long |
| 111 | 010 | d16(PC) | PC-relative displacement |
| 111 | 011 | d8(PC,Xn) | PC-relative index |
| 111 | 100 | #imm | Immediate |

### Effective Address Resolution

```csharp
uint ResolveEA(int mode, int reg, int size)
{
    return (mode, reg) switch
    {
        (0, _) => /* data register — return register index, not address */,
        (1, _) => /* address register — return register index */,
        (2, _) => A[reg],
        (3, _) => PostIncrement(reg, size),
        (4, _) => PreDecrement(reg, size),
        (5, _) => (uint)(A[reg] + (short)FetchWord()),
        (6, _) => ResolveIndexed(A[reg]),
        (7, 0) => (uint)(short)FetchWord(),           // sign-extend
        (7, 1) => FetchLong(),
        (7, 2) => (uint)(PC + (short)FetchWord()),
        (7, 3) => ResolveIndexed(PC),
        (7, 4) => FetchImmediate(size),
        _ => throw new IllegalInstructionException()
    };
}
```

## Cycle Counting

### Bus Cycle Model

Each bus access takes 4 clock cycles minimum. The 68000 uses a 2-word prefetch queue.

```csharp
int cycles;  // Accumulated cycles for current instruction

void ReadWord(uint addr)  => cycles += 4;  // S4-S7 bus cycle
void WriteWord(uint addr) => cycles += 4;  // S4-S7 bus cycle
void InternalCycles(int n) => cycles += n; // Internal processing
```

### Instruction Timing Examples

| Instruction | Timing | Breakdown |
|-------------|--------|-----------|
| MOVE.L Dn,Dn | 4 cycles | 1 read (prefetch) |
| MOVE.W (An),(An) | 12 cycles | 2 reads + 1 write |
| ADD.L #imm,Dn | 16 cycles | 3 reads + internal |
| MULU Dn,Dn | 38-70 cycles | Depends on operand bits |
| DIVS Dn,Dn | 120-158 cycles | Complex microcode |
| MOVEM.L regs,-(An) | 8+8n cycles | 8 base + 8 per register |

## Exception Processing

### Exception Sequence (common to all)

1. Copy SR to internal temp
2. Set S bit (enter supervisor), clear T bit (disable trace)
3. If interrupt: set interrupt mask to current level
4. Push PC (long) to supervisor stack
5. Push saved SR (word) to supervisor stack
6. Read exception vector from table
7. Load PC from vector
8. Prefetch first instruction words

### Exception Priorities

1. Reset (highest)
2. Address Error
3. Bus Error
4. Trace
5. Interrupt (level 7 NMI non-maskable, levels 1-6 masked by SR)
6. Illegal Instruction
7. Privilege Violation
8. TRAP instructions

### Interrupt Handling

```csharp
void CheckInterrupts()
{
    int pending = GetHighestPendingInterrupt(); // from Paula INTREQ
    int mask = (SR >> 8) & 7;

    if (pending > mask || pending == 7) // Level 7 is NMI
    {
        ProcessException(24 + pending); // Auto-vector: $64 + level*4
    }
}
```

## Implementation Notes

### Prefetch Queue

The 68000 has a 2-word prefetch queue. For accurate emulation:
- IRC: Instruction Register Capture (next word to be decoded)
- IRD: Instruction Register Decode (current instruction word)
- Prefetch happens during execute phase of previous instruction

### Address Error

Triggered when word/long access at odd address. Must push:
- Access address, instruction register, status register
- Function code (read/write, instruction/data)

### Special Considerations for Amiga

- CPU clock: 7.09 MHz PAL, 7.16 MHz NTSC
- Bus contention with Agnus DMA: CPU may be delayed waiting for bus
- STOP instruction: CPU halts until interrupt (used by OS idle loop)
- RESET instruction: asserts RESET line for 124 clock cycles (resets peripherals, not CPU)
